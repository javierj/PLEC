Prototipado rápido de aplicaciones mediante LEGO® y Componentes Web (PLEC)

Abstract

La interfaz gráfica de usuario o GUI es probablemente el elemento de un sistema de software más visible para los usuarios finales. Por lo tanto, a los usuarios finales les preocupa mucho cómo se crea la GUI. Incluso, un usuario puede medir la calidad de una aplicación solo por su GUI. Las GUI también son un punto de encuentro donde los participantes técnicos y no técnicos se reúnen entre sí. Los usuarios finales no discuten el diseño de la capa de acceso a datos ni la implantación de una arquitectura distribuida, pero el usuario final puede ser muy crítico con la GUI. Este comportamiento impone una forma diferente de trabajar con GUI. En el desarrollo de GUI, la gestión de ideas, necesidades y expectativas de los usuarios finales son más importantes que otras partes del sistema. La comunicación fallida es a menudo la raíz de una especificación de requisitos inadecuada. Por lo general, los desarrolladores entrevistan a clientes y partes interesadas para descubrir la funcionalidad del sistema y los requisitos para la GUI. Después de un tiempo, los clientes y las partes interesadas ven lo que los desarrolladores creen que es la implementación correcta de lo que los desarrolladores entendieron de las entrevistas. Además, tal vez el cliente no recordó la entrevista o incluso el contexto del cliente es diferente. Este proyecto propone un cambio de paradigma. En lugar de hablar con el cliente, PLEC propone trabajar con clientes y partes interesadas para desarrollar un prototipo de GUI en un taller. Después de eso, los desarrolladores y las partes interesadas comparten la misma visión del sistema GUI y los desarrolladores pueden trabajar alineados con sus clientes. El objetivo de este documento es introducir un proceso, llamado PLEC, para obtener un prototipo de GUI con la participación activa de los usuarios y otras partes interesadas no técnicas. El objetivo de este proceso es primero, ayudar a los usuarios a pensar y decidir qué GUI quieren para su sistema y, segundo, tener un prototipo visual que el usuario pueda validar.
